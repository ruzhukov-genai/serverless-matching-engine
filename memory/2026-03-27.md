### Key Design Decisions

- **OnceCell singleton**: `WorkerState` is initialized once on first request, reused across Lambda invocations (warm starts)
- **Cold start latency**: pairs_cache loaded from PG on cold start (~5ms per worker notes)
- **Stateless constraint preserved**: No in-memory order book replicas, all mutable state in Valkey/PG
- **Lua matching untouched**: ADR-004 enforced — matching uses `cache::match_order_lua`, no changes to logic
- **No cross-invoke overhead**: Gateway directly calls `crate::worker::process_order_inline()`

### Architecture Impact

- **Before**: Client → API GW → Gateway Lambda (validate) → cross-invoke → Worker Lambda (match + persist) → response
- **After**: Client → API GW → Gateway Lambda (validate + inline match + persist) → response

Eliminates the ~800ms Lambda-to-Lambda invoke round-trip, giving true E2E latency measurement from `received_at` to `persisted_at` inside a single Lambda invocation.

### Constraints Preserved

- ✅ ADR-001: Workers stateless (pairs_cache loaded cold, not cached warm)
- ✅ ADR-004: Matching uses Lua EVAL atomically (no changes)
