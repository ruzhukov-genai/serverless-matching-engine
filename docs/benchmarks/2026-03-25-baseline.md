# Baseline Benchmarks — 2026-03-25

**Run ID:** `bench-20260325-2102`
**Environment:** Local dev (Ubuntu 24.04, Docker Compose)
**Infrastructure:** Valkey 8.1 + PostgreSQL 16 (Docker containers)
**Build:** Release, fresh volumes, seeded liquidity
**Duration:** 60s per concurrency level

## Benchmark Configuration

- **Pairs:** BTC-USDT
- **Order pattern:** Alternating buy/sell at 70600/70800 (GTC limit orders)
- **Order size:** 0.001 BTC per order
- **Seeded liquidity:** 100 passive orders (10 levels × 5/side) + 40 active orders at crossing prices
- **client_order_id tagging:** `bench-YYYYMMDD-HHMM-n{clients}-c{client_id}-{seq}`
- **DB truncate between levels:** yes

## Key Observations

1. **Linear scaling** — throughput scales from 1.9/s to 192.2/s (100x clients → 100x throughput)
2. **Zero errors** across all concurrency levels
3. **Lua matching is constant** — dispatch p50 stays at 2-5ms regardless of load
4. **PG persist is the tail driver** — p99 reaches 163ms at c=100
5. **~50% match rate** — alternating buy/sell pattern, as expected
6. **No saturation point hit** — c=100 still scaling linearly
